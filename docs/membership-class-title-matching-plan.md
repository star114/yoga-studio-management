# 회원권-수업명 매칭 개선 플랜

## 배경
- 현재 추천 수업 조회, 수업 예약, 출석 처리에서 회원권 이름과 수업명을 사실상 정확히 같은 문자열로 비교한다.
- 그래서 `아침요가 3개월`, `아침요가 1회권`, `아침요가 + 설명` 같은 회원권 이름은 `아침요가` 수업과 자동 매칭되지 않는다.

## 목표
- DB 스키마 변경 없이 회원권 이름 뒤에 기간, 횟수, 설명이 붙은 경우를 자동 매칭한다.
- 추천 수업 조회, 수업 예약, 출석 처리에서 동일한 매칭 규칙을 사용한다.
- 기존에 회원권과 수업이 매칭되지 않아도 관리자가 승인하면 예약/출석 처리가 가능했던 흐름을 유지한다.

## 비목표
- 회원권 종류에 별도 매칭 필드를 추가하지 않는다.
- 관리자 UI나 회원권 생성 화면 구조는 변경하지 않는다.
- 복수 수업명을 한 회원권에 매핑하는 설계까지 확장하지 않는다.

## 매칭 규칙
1. 회원권명과 수업명을 공백 정규화한다.
2. 정규화 후 완전히 같으면 매칭으로 본다.
3. 회원권명이 수업명으로 시작하고, 뒤에 이어지는 첫 글자가 `문자(letter)`가 아니면 매칭으로 본다.

## 예시
- 매칭
  - `아침요가` -> `아침요가`
  - `아침요가 3개월` -> `아침요가`
  - `아침요가 1회권` -> `아침요가`
  - `아침요가 + 설명` -> `아침요가`
  - `아침요가(주3회)` -> `아침요가`
- 비매칭
  - `아침요가심화` -> `아침요가`
  - `저녁아침요가` -> `아침요가`

## 구현 범위
- 백엔드 공통 helper 추가
- 추천 수업 조회의 매칭 및 페이지네이션 반영
- 수업 예약 시 회원권 매칭 검증 반영
- 출석 등록 시 자동 회원권 선택 순서 반영
- 관련 unit/e2e 테스트 보강

## 기존 승인 흐름 보존
- 자동 매칭 범위가 넓어져도, 그래도 미매칭인 경우에는 기존처럼 관리자 승인 플로우로 떨어져야 한다.
- `allow_cross_membership_registration`가 없으면 기존과 동일하게 `CROSS_MEMBERSHIP_CONFIRM_REQUIRED`를 반환한다.
- `allow_cross_membership_registration`가 있으면 기존처럼 대체 회원권으로 예약 가능해야 한다.
- 출석 처리에서 자동 매칭 회원권이 없을 때도 기존처럼 최신 활성 회원권 fallback 동작은 유지한다.

## 리스크
- 규칙이 너무 느슨하면 의도치 않은 수업명이 매칭될 수 있다.
- 추천 수업 조회를 애플리케이션 레벨에서 필터링하면 쿼리 결과 수가 많을 때 비용이 늘 수 있다.
- 세 경로가 같은 helper를 쓰지 않으면 다시 규칙이 어긋날 수 있다.

## 완료조건
- `아침요가 3개월`, `아침요가 1회권`, `아침요가 + 설명` 회원권으로 `아침요가` 수업 추천/예약 가능
- `아침요가심화`는 `아침요가`와 자동 매칭되지 않음
- 관리자 승인 기반 교차 회원권 예약 흐름 유지
- 출석 처리 fallback 흐름 유지
- backend `lint`, `build`, `test:unit`, `test:e2e`, `test:coverage` 통과
