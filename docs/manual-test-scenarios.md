# 수동 테스트 시나리오

## 목적
- 현재 프로젝트의 관리자/고객/운영 기능을 브라우저와 터미널에서 수동으로 검증하기 위한 기준 문서
- 릴리즈 전 smoke test, 기능 변경 후 regression test, 운영 배포 후 sanity check에 공통 사용

## 범위
- 현재 라우트에 연결된 화면 기능
- 운영 스크립트: `start.sh`, `start-local.sh`, `deploy.sh`, `auto-update.sh`, `backup.sh`, `restore.sh`
- 권한/로그인/주요 데이터 흐름

## 범위 제외
- 자동화 테스트가 이미 충분히 커버하는 내부 구현 상세
- 성능 테스트, 부하 테스트, 보안 침투 테스트

## 권장 실행 순서
1. 공통 사전 점검
2. 인증/권한
3. 관리자 핵심 플로우
4. 고객 핵심 플로우
5. 운영 스크립트

운영 규칙:
- UI/라우트 변경은 해당 화면 시나리오를 최소 세트 이상 수행한다.
- API만 바꿨더라도 사용자 플로우에 영향이 있으면 연결된 화면 시나리오를 함께 수행한다.
- 운영 스크립트 변경은 터미널 기준 수동 검증 결과를 PR에 남긴다.

## 테스트 환경
- 브라우저: Chrome 계열 1개, Safari 또는 Firefox 1개 권장
- 해상도: Desktop 1440px 이상, Mobile width 390px 전후
- 서버 실행 방식 중 하나
  - 로컬 Docker: `./start.sh`
  - 로컬 개발 모드: `./start-local.sh`
  - 운영형 Docker: `./deploy.sh`

## 권장 테스트 데이터
- 관리자 계정 1개
  - 예: `admin / admin123`
- 고객 계정 1개 이상
  - 예약 수업이 있는 고객 1명
  - 출석 기록이 6건 이상인 고객 1명
  - 결석 기록이 있는 고객 1명
- 회원권 데이터
  - 무제한 또는 잔여 횟수 넉넉한 회원권 1개
  - 잔여 횟수 0인 회원권 1개
  - 동일 이름의 지급 회원권 2개 이상인 케이스 1개
- 수업 데이터
  - 접수중 수업
  - 완료 수업
  - 예약자 있는 수업
  - 출석자 있는 수업
  - 결석 처리된 수업

## 공통 체크포인트
- 로딩 스피너에서 정상 화면으로 전환된다.
- API 실패 시 화면이 완전히 깨지지 않고 에러 문구 또는 fallback 상태가 보인다.
- 버튼 disabled 상태가 실제 동작 제한과 일치한다.
- 날짜/시간 표기가 한국어 형식으로 자연스럽다.
- 브라우저 콘솔에 치명적 오류가 없다.

## 1. 인증 / 권한

### AUTH-01 관리자 로그인
- 사전조건: 관리자 계정 존재
- 절차:
  1. `/login` 접속
  2. 관리자 아이디/비밀번호 입력
  3. 로그인 클릭
- 기대결과:
  - `/`로 이동
  - 관리자 대시보드가 열린다.
  - 상단 메뉴에 `고객 관리`, `회원권 관리`, `수업 관리`, `관리자 계정`이 보인다.

### AUTH-02 고객 로그인
- 사전조건: 고객 `login_id`와 비밀번호 존재
- 절차:
  1. `/login` 접속
  2. 고객 아이디/비밀번호 입력
  3. 로그인 클릭
- 기대결과:
  - `/`로 이동
  - 고객 대시보드가 열린다.
  - 상단 메뉴에 `수련 기록`, `회원권`, `내 정보`가 보인다.

### AUTH-03 로그인 실패
- 절차:
  1. 존재하지 않는 계정 또는 틀린 비밀번호 입력
  2. 로그인 클릭
- 기대결과:
  - 로그인 화면 유지
  - 실패 메시지 표시

### AUTH-04 권한 우회 차단
- 절차:
  1. 고객으로 로그인
  2. `/customers`, `/classes`, `/admin-accounts` 직접 접근
- 기대결과:
  - 관리자 전용 페이지가 열리지 않는다.
  - `/` 또는 접근 가능한 화면으로 리다이렉트된다.

## 2. 관리자 대시보드

### ADM-DASH-01 요약 카드와 최근 고객 표시
- 절차:
  1. 관리자 로그인 후 `/` 진입
- 기대결과:
  - 전체 회원 수, 오늘 출석 수가 보인다.
  - 최근 등록 회원 카드가 보인다.

### ADM-DASH-02 수업 캘린더 월/주/일 전환
- 절차:
  1. `월간`, `주간`, `일간` 버튼을 차례로 클릭
  2. `오늘`, `이전`, `다음` 버튼도 각각 클릭
- 기대결과:
  - 각 뷰가 정상 전환된다.
  - 헤더 날짜 범위가 현재 뷰에 맞게 바뀐다.

### ADM-DASH-03 캘린더 상태 배지 확인
- 절차:
  1. 접수중/진행중/완료/마감 상태 수업이 있는 날짜를 확인
- 기대결과:
  - 상태가 글자와 색으로 구분된다.
  - 범례와 실제 카드 표현이 일관된다.

### ADM-DASH-04 캘린더에서 수업 상세 이동
- 절차:
  1. 수업이 있는 날짜/카드를 클릭
- 기대결과:
  - 해당 수업 상세 페이지로 이동한다.

## 3. 고객 관리

### ADM-CUST-01 고객 목록 조회와 검색
- 절차:
  1. `/customers` 이동
  2. 검색어 입력
- 기대결과:
  - 고객 목록이 보인다.
  - 이름/전화번호 기준 검색 결과가 필터링된다.

### ADM-CUST-02 고객 생성
- 절차:
  1. 고객 생성 폼에 이름/전화번호 입력
  2. 저장
- 기대결과:
  - 고객이 생성된다.
  - 전화번호가 표준 형식으로 저장된다.

### ADM-CUST-03 고객 생성 실패
- 절차:
  1. 전화번호 없이 저장 또는 중복/유효성 오류를 유도
- 기대결과:
  - 에러 메시지 표시
  - 잘못된 데이터가 목록에 추가되지 않는다.

### ADM-CUST-04 고객 삭제
- 절차:
  1. 고객 삭제 클릭
  2. 취소, 재시도 후 확인
- 기대결과:
  - 취소 시 삭제되지 않는다.
  - 확인 시 목록에서 제거된다.

## 4. 고객 상세

### ADM-CUST-DETAIL-01 기본 정보 조회/수정
- 절차:
  1. 고객 상세 진입
  2. `기본 정보 수정`
  3. 이름/전화번호/메모 수정 후 저장
- 기대결과:
  - 수정 결과가 즉시 반영된다.
  - 빈 값/잘못된 전화번호는 저장되지 않는다.

### ADM-CUST-DETAIL-02 고객 비밀번호 초기화
- 절차:
  1. `비밀번호 초기화`
  2. 취소 후 재시도
  3. 확인
- 기대결과:
  - 취소 시 변경 없음
  - 확인 시 기본값 `12345`로 초기화 성공 메시지 또는 정상 완료 동작

### ADM-CUST-DETAIL-03 지급된 회원권 목록 확인
- 절차:
  1. 고객 상세의 `지급된 회원권` 섹션 확인
- 기대결과:
  - 회원권명, 활성 여부, 예약 가능 잔여, 소진 횟수, 시작일, 예상 종료일이 보인다.

### ADM-CUST-DETAIL-04 회원권 지급
- 절차:
  1. 회원권 종류 선택
  2. 필요 시 메모 입력
  3. `회원권 지급`
- 기대결과:
  - 지급된 회원권 목록에 즉시 반영된다.

### ADM-CUST-DETAIL-05 회원권 수정
- 절차:
  1. 기존 지급 회원권의 `수정`
  2. 잔여 횟수/메모/활성 상태 수정
  3. 저장 후 취소 흐름도 확인
- 기대결과:
  - 수정 값 반영
  - 취소 시 원래 값 유지

### ADM-CUST-DETAIL-06 회원권 삭제 또는 비활성화
- 사전조건: 삭제 가능한 회원권과 비활성화만 가능한 회원권이 각각 존재
- 절차:
  1. 두 유형 모두 삭제/비활성화 버튼 동작 확인
- 기대결과:
  - 사용/예약 이력 없는 회원권은 삭제 가능
  - 이미 사용했거나 예약 연결된 회원권은 비활성화 흐름으로 유도

### ADM-CUST-DETAIL-07 추천 수업 빠른 예약
- 사전조건: 예약 가능한 추천 수업 존재
- 절차:
  1. 회원권 카드에서 `불러오기`
  2. 추천 수업 목록 확인
  3. 빠른 예약 실행
- 기대결과:
  - 해당 추천 항목만 예약 처리된다.
  - 예약 후 고객 수업 기록에 `예약`으로 보인다.

### ADM-CUST-DETAIL-08 교차 회원권 예약 확인
- 사전조건: 같은 이름 회원권이 없고 대체 회원권으로 예약 가능한 수업 존재
- 절차:
  1. 빠른 예약 시도
  2. 교차 회원권 확인 다이얼로그 확인
  3. 확인/취소 각각 수행
- 기대결과:
  - 취소 시 예약되지 않음
  - 확인 시 예약 성공

### ADM-CUST-DETAIL-09 수업 기록 필터/페이지네이션
- 절차:
  1. `필터` 열기
  2. 상태, 기간 조건 적용
  3. 페이지 번호/이전/다음 이동
- 기대결과:
  - `출석/예약/결석`이 조건에 맞게 필터링된다.
  - 페이지네이션이 정상 동작한다.

### ADM-CUST-DETAIL-10 수업 기록에서 예약 취소/결석 처리
- 절차:
  1. `예약` 항목에서 `예약 취소`
  2. `출석` 항목에서 `결석 처리`
- 기대결과:
  - 상태가 즉시 변경된다.
  - 실패 시 에러 메시지 표시

### ADM-CUST-DETAIL-11 수업 기록에서 수업 상세 이동
- 절차:
  1. `class_id`가 있는 수업 기록 제목 클릭
- 기대결과:
  - 해당 수업 상세로 이동한다.
